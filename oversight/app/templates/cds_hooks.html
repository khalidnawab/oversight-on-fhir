CDS Hooks is the HL7 standard for surfacing decision support inside the EHR workflow. Oversight-on-FHIR exposes the agent's de-escalation recommendation as a CDS Hooks service, so an EHR can show it as a card at the point of care — with the AI disclosure and an accept / edit / reject control that feeds the same FHIR oversight capture. This completes the standards-native path: FHIR for data, SMART for auth, CDS Hooks for delivery.

Recommendation → card mapping

RecommendationCard
surface · narrowinfo summary + rationale, with an accept suggestion whose uuid is the GuidanceResponse reference
escalatewarning escalation summary + triggered high-risk rule; no suggestion (a human must decide)

Selecting the card's suggestion (accept) or overriding it maps to the same oversight-capture path as the clinician UI — producing an AuditEvent (OversightEvent profile) referencing the reviewed recommendation, with a coded disposition and reason.
